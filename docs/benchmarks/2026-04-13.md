# Search Quality Benchmark: 2026-04-13

## PR #64 Impact Analysis

Comparing search quality before and after the search quality boost (compiled truth
ranking, RRF normalization, source-aware dedup). Measured against 5 seeded brain
pages with 10 chunks total, using 8 benchmark queries with structured mock embeddings.

Inspired by [Ramp Labs' "Latent Briefing" paper](https://ramp.com) (April 2026).

## Key Finding

**The compiled truth boost works as intended for entity lookups but over-corrects
for temporal/event queries.** When you ask "when did we last meet Pedro?", the 2.0x
boost forces the compiled truth chunk to rank first instead of the timeline entry
that actually answers the question.

P@1 and MRR are unchanged (1.0 both ways) because the correct PAGE is always found.
The issue is within-page chunk ordering: source accuracy drops from 100% to 71.4%
because the boost overrides the natural ranking for 2 of 7 queries.

**This validates the `--detail` parameter.** Agents should use `--detail low` for
"who is X?" queries (compiled truth only) and `--detail high` for "when did we last
meet X?" queries (all chunks, natural ranking). The default `medium` with the boost
is the right compromise for most queries, but the agent needs to use the knob.

**The boost is not wrong, it's blunt.** The next improvement is query intent
classification (deferred TODO) where the system automatically selects the right
detail level based on the query. Until then, the agent makes the call.

## Methodology

- **Engine:** PGLite (in-memory, Postgres 17.5 via WASM)
- **Pages:** 5 test pages (2 person, 2 company, 1 concept) with 2 chunks each
- **Embeddings:** Structured basis vectors with blending (deterministic cosine distances)
- **Queries:** 7 with ground truth + 1 negative control
- **Baseline:** Standard RRF fusion (K=60), no normalization, no source boost
- **PR #64:** RRF normalized to 0-1, 2.0x compiled_truth boost, source-aware dedup
- **Metrics:** P@1 (precision at rank 1), MRR (mean reciprocal rank), nDCG@5, source accuracy

## What Changed in PR #64

1. **RRF normalization** — scores normalized to 0-1 before boosting
2. **Compiled truth boost** — 2.0x multiplier for compiled_truth chunks
3. **Source-aware dedup** — guarantees compiled truth chunk per page in results
4. **Cosine re-scoring** — blends RRF + cosine similarity (0.7/0.3) before dedup
5. **Detail parameter** — `--detail low/medium/high` controls timeline inclusion
6. **CJK word count** — fixes query expansion for Chinese/Japanese/Korean
7. **Eval harness** — `gbrain eval` with P@k, R@k, MRR, nDCG@k + A/B comparison
